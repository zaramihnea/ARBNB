1. Introduction

ARBNB is a conceptual design for a property rental platform, modeled after the core ideas behind Airbnb. This documentation outlines the interaction patterns, design principles, and user experience decisions as implemented in a high-fidelity Figma prototype. The prototype serves as a foundation for future development and user testing.

2. Interaction Design Patterns

2.1 Search & Filter

Users engage with a dynamic search bar that supports keyword inputs and autocomplete suggestions. This model follows the Search and Refine pattern and is rooted in the heuristic of visibility of system status. This pattern enhances discoverability and ensures users can locate relevant items quickly by narrowing down options.

Example: On the home screen of the mobile app, users begin typing a city name such as “Cluj-Napoca” into the search bar. Instantly, the interface displays location suggestions beneath the input field, including matching neighborhoods and popular tourist districts. The property list updates dynamically with each filter adjustment, ensuring that only the most relevant options are shown.

2.2 Progressive Disclosure

Only the most essential information is shown initially, while additional details are hidden and can be revealed on user action (e.g., click, tap). This reduces cognitive load and avoids overwhelming users. This supports the principle of cognitive economy and aligns with the progressive disclosure pattern.

Example: Property listings show a short description, and clicking expands to reveal the full details, like host information(name, hosting history) description of the property, number of guests, bedrooms, amenities, house rules, price, address or general area with a map preview.

2.3 Infinite Scroll / Load More

As users scroll down a list, additional content is automatically loaded, creating a seamless and uninterrupted browsing experience, improving navigation flow and reducing the need for pagination or extra clicks. Alternatively, a “Load More” button can be used to load the next batch of results.

Example: The Property Listing page showcases all available accommodations based on the user’s input, giving a broad overview of relevant options in a visually engaging, scrollable format. Each property is displayed as a card featuring a thumbnail image, title, rating, and information abot the host. This layout allows users to quickly scan through listings without needing to open each one individually. As users scroll down the screen, the interface dynamically loads additional entries using infinite scroll, ensuring a seamless browsing experience without the need for manual pagination. This approach enhances user engagement by minimizing friction, keeping attention focused on discovering new properties without interruption.

2.4 Tabbed Navigation

Information is organized into distinct tabs that users can quickly switch between without leaving the current page. This helps in segmenting content logically.

Example: On a property page, tabs like “Reviews”, "Details", “Map”, and “Tour” let users access different content views easily. This tabbed navigation structure helps prevent information overload by organizing content into distinct, easily accessible sections. For instance, users can quickly switch to the “Reviews” tab to read guest feedback without losing their place, or jump to the “Map” tab to view the property's location and nearby attractions. The interface updates smoothly without reloading the page, ensuring a seamless and efficient user experience while maintaining context.

2.5 Breadcrumb Navigation

Breadcrumbs display the user’s navigation path in a hierarchical format, allowing them to easily backtrack through previous levels or sections of the app. This pattern is particularly effective in mobile interfaces with nested or categorized content, where users may dive deep into a specific item or category. Breadcrumbs enhance usability by reinforcing the user’s current position within the app structure and providing a clear route to return to broader overviews without needing to use the back button multiple times.

Example: A navigation trail such as Home > Cabins > Rustic Cabin in Cluj-Napoca, Romania > Reviews allows the user to tap on “Cabins” to return to the general category or on “Cluj-Napoca” to view all cabins available in that specific region. This supports quick backward navigation and helps users efficiently move through different content layers.

3. Design Decisions

3.1 Visual Minimalism

For this prototype, a modern and minimalistic UI style was chosen to create a clean and intuitive user experience. The design emphasizes clear typography, generous whitespace, and a calm, neutral color palette. These choices aim to prioritize the display of essential content while minimizing visual clutter, which helps reduce user cognitive load and decision fatigue. Since this is a Figma prototype, the focus was on establishing a clear visual hierarchy and an aesthetic foundation that can be iterated on during development.

3.2 Role-based Navigation

The navigation structure in the prototype adapts dynamically based on the user’s role — whether they are a host or a guest. Hosts have access to features like property listing management and analytics dashboards, while guests see personalized recommendations and trip planning tools. Although this behavior is conceptual within the prototype, it reflects an important design decision aimed at tailoring the experience to different user needs and simplifying interface complexity by showing only relevant features.

3.3 Trust Signals

Building user trust is vital for a platform facilitating bookings and payments. The prototype integrates visual trust elements such as user reviews. These signals are prominently displayed to increase user confidence and reduce hesitation during the booking process. While the prototype does not implement backend verification, these design elements serve as a blueprint for how trust and security can be visually communicated.

3.4 Performance Considerations

Although performance cannot be fully tested in a static Figma prototype, design decisions anticipate smooth and responsive user interactions. The prototype incorporates concepts like lazy loading placeholders for images, modular component layouts, and streamlined data presentation. These concepts guide future development toward efficient rendering and data fetching strategies, ensuring the final app remains performant even on constrained mobile networks.

4. User Manual

4.1 Getting Started

  • Navigate to Figma Prototype – ARBNB
  • Click Sign Up and enter your details to create an account.
  • Verify your account via email before logging in.

4.2 Browsing & Booking a Stay (Guests)

You can browse and search for places without needing to create an account.

  1. Search by city or region and select check-in/check-out dates.
  2. Apply filters for price, amenities, and property type.
  3. Click a listing for full details.
  4. Note: To book a stay, you must be logged in. If you don’t have an account, you will be prompted to sign up or log in at checkout.
  5. Input guest details and payment info, then confirm reservation.
  6. Access your upcoming trips via the Trips tab once logged in.

4.3 Hosting a Property

Hosting requires an account. You must be logged in to list a property for rent.

  1. Log in and navigate to Host Dashboard.
  2. Click New Listing and complete the property form.
  3. Upload high-quality photos and set pricing rules.
  4. Submit for review; once approved, the property becomes public.
  5. Track bookings, earnings, and messages via the dashboard.

4.4 Managing Your Account

  • Edit personal info and upload a profile picture under Settings.
  • Enable two-factor authentication for enhanced security.
  • Manage payout methods and notification preferences.

5. Prototype Details

The current version of ARBNB is a fully interactive Figma prototype designed to simulate key user flows for both guests and hosts. This includes:

  • Onboarding and registration flow
  • Property search and filtering interactions
  • Listing details and booking interface
  • Host dashboard and listing creation

The prototype was designed using auto layout, variants, and component libraries to ensure consistency and responsiveness across screens. No backend functionality is implemented at this stage.

You can view or interact with the prototype at: Figma Prototype – ARBNB

6. Screen Overview

The ARBNB prototype consists of a series of interconnected screens that guide both guests and hosts through essential actions such as browsing, booking, and managing listings. Below is a summary of each screen and its role in the user flow:

6.1 Home Screen

The Home Screen is the primary landing page for users and features a central search bar for destination input. Directly below the search bar, a horizontally scrollable category menu allows users to quickly filter listings by popular property types or themes:

  • Cabins
  • Lakefront
  • A-Frames
  • Tiny Homes
  • National Parks

These categories enable users to explore curated listings suited to their travel preferences without needing to type specific queries.

Below the category menu, a section displays Recommended Locations tailored to the user's interests and popular trends. Users can browse and select these recommendations to quickly access curated listings and further details about the destination.

At the bottom of the screen, a persistent navigation bar offers quick access to the app’s main sections:

  • Home: Returns to this landing page from anywhere in the app.
  • My Trips: Displays the user's current and past bookings.
  • Map: Opens an interactive map view of available listings.
  • Profile: Allows users to manage account settings and personal preferences.
  • VR: Provides virtual reality tours of selected properties.

6.2 Item Page (Listing Detail View)

The Item Page is accessed when a user selects a listing from the home screen, search results, or recommended locations. This screen provides a comprehensive overview of a specific property and serves as the main decision point before booking.

The top of the page typically displays a gallery of property images, followed by structured content organized into sections. Key information includes:

  • Property Name
  • User rating
  • Host Information: Name, hosting history.
  • Place Overview: Description of the property, number of guests, bedrooms, amenities, and house rules.
  • Price: Nightly rate and total estimated cost based on selected dates (if applicable).
  • Location: Address or general area with a map preview.

A secondary navigation bar is located beneath the main gallery, allowing users to easily switch between detailed sections:

  • Reviews: User feedback, star ratings, and guest comments.
  • Details: Full property description, amenities, check-in/out policies, and rules.
  • Map: A larger interactive map showing the property's location.
  • Tour: Option to view a virtual tour or immersive walkthrough of the listing (when available).

The design of this page emphasizes clarity and transparency to help users make informed decisions and feel confident in their booking.

6.3 Search Screen

The Search Screen allows users to browse available listings based on search input and filters. This screen is typically accessed after a user initiates a search by entering a destination. It serves as the central hub for exploring available properties that match the user's criteria.

The top of the screen displays a persistent search bar with current query parameters (e.g., destination), allowing users to quickly refine or change their search.

The main body of the screen displays a vertically scrollable list of matching results. Each listing includes a preview card with:

  • Listing Image: A representative thumbnail showcasing the property.
  • Property Title: The name or headline assigned to the listing by the host.
  • User Rating: Average rating based on guest reviews, typically displayed with stars and the number of reviews.
  • Host Information: Includes the host’s name and hosting history (e.g., the length of time they have been active on the platform).
  • Availability Period: The range of dates during which the property is available for booking.

The Search Screen focuses on discoverability, allowing users to narrow down a large set of options into listings that fit their preferences and constraints.

6.4 Map Screen

The Map Screen provides a geographic visualization of available listings, enabling users to explore accommodations in relation to their current location or a destination of interest. It enhances spatial decision-making and offers an intuitive, location-first browsing experience.

At the top of the screen, a persistent search bar allows users to manually input a desired location or address. Upon submitting a query, the map automatically recenters and refreshes the visible listings based on the new area.

At the bottom of the screen, a horizontally scrollable recommendation carousel showcases handpicked or trending listings. Each card displays a preview image, property name, and the distance from the user's current location. Tapping a card recenters the map and highlights the corresponding listing pin, allowing users to explore properties both visually and spatially.

The Map Screen is particularly helpful for users who prioritize proximity to specific areas—such as tourist landmarks, public transport hubs, or natural attractions—when selecting their accommodation.

6.5 Profile Screen

The Profile Screen serves as the central hub for user account management and application preferences. It allows users to access personal information, security settings, and support resources.

At the top of the screen, the user’s personal information is prominently displayed, including:

  • Profile Picture
  • Full Name
  • Username

The central portion of the screen presents a series of actionable settings and account management options:

  • My Account: View and edit personal information such as email, phone number, and password.
  • Saved Beneficiary: Manage saved payment recipients or emergency contacts, if applicable.
  • Face ID / Touch ID: Enable or disable biometric authentication for quicker and more secure access.
  • Two-Factor Authentication: Configure additional security measures using SMS or authenticator apps.
  • Log Out: Securely log out of the account.

Below the main account section, users can access general information and support features:

  • Help and Support: Access FAQs, contact customer service, or troubleshoot issues.
  • About App: View application version, developer information, and acknowledgments.
  • Privacy Notice: Review the app’s data handling and user privacy policy.
  • Terms of Service: Read the legal terms and conditions for using the ARBNB platform.

The Profile Screen is designed with clarity and accessibility in mind, ensuring that all account-related features and legal resources are easy to locate and navigate.

6.6 Virtual Reality (VR) Experience Screen

The VR Experience Screen provides an immersive, interactive 2D environment that allows users to explore a selected property in virtual reality. This feature simulates a real-world tour, enhancing user confidence and engagement when evaluating a potential accommodation.

A key element of the VR interface is the True-Scale Luggage Organizer, a virtual object that users can position within the space to compare dimensions and better understand room sizes. This functionality supports practical decision-making regarding space adequacy for travel needs.

Throughout the VR environment, users can interact with a series of smart icons that reveal contextual information or enable virtual controls. These include:

  • Wi-Fi Network Status: Displays real-time connection details such as:
    • Network Name (SSID)
    • Signal Strength
    • Number of Connected Devices
    • Guest Network Availability
  • Coffee Machine: Provides step-by-step instructions for using the appliance, simulating real-life interaction for user familiarity before check-in.
  • Climate Control: Enables virtual adjustment of temperature settings, including increasing or decreasing the room’s temperature through intuitive VR controls.
  • Smart Lamp Control: Allows users to modify the brightness level of the room's smart lighting via a responsive slider interface or gesture-based interaction.
  • Hidden Safe: Introduces a secure element of the room tour, where users can simulate unlocking a digital safe using one of three options:
    • Facial Recognition
    • Fingerprint Scan
    • Manual Passcode Entry

This screen enhances transparency, builds trust, and sets user expectations by offering a digital twin of the real accommodation. It also aligns with smart-home trends by integrating interactive, simulated devices within the virtual space.
